## 주요 변경 사항 (feature/front 브랜치)

### 2. 프론트엔드 전면 개편 (`frontend/`)
- `app.js`, `index.html`, `style.css` 대규모 수정 (1,128줄 추가, 556줄 삭제)
- 가중치별 루트 선택 기능 추가
- 길찾기 오류 수정

### 3. K-Means 쾌적도 분류를 A* 라우팅에 통합 (`backend/route_model.py`)
- 기존 독립 스크립트(`ai_comfort_clustering.py`)였던 K-Means를 실제 라우팅 파이프라인에 통합
- Dijkstra → A* 전환 (Haversine 기반 분 단위 휴리스틱으로 shadow_weight 단위 일치)
- 피처: shadow_percent, sunny_exposure(직사광선 노출 거리), is_footway
- StandardScaler 정규화 적용 (피처 스케일 왜곡 방지)
- Silhouette Score 기반 최적 K 자동 선택 (K=2~5)
- 등급별 가중치 배율을 K에 맞춰 선형 보간

### 4. 문서 최신화 (`README.md`, `공공데이터개발기획서_작성내용.md`)
- 기술 스택에서 Scikit-learn(K-Means) 단독 표기 제거 → A* 라우팅 내 통합으로 반영
- OSMnx 도로망을 정적 데이터 → 런타임 동적 다운로드로 정정
- 핵심 라우팅 로직 순서 실제 코드 기준으로 수정
