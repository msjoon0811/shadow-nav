# 2026년 전국 통합데이터 활용 공모전 서비스 개발 기획서

**서비스명: Shadow-Nav (그늘길 내비게이션)**

---

## 1) 개발 결과물 개요

폭염 및 기후위기 심화에 대응하여, 보행자와 자전거 이용자에게 **시간대별 실제 건물 그림자를 반영한 가장 시원한(그늘진) 최적 경로**를 제공하는 스마트 웹 내비게이션 서비스입니다.

공공데이터(3D 건물 정보, 실시간 신호등, 공영자전거 대여소)와 AI 기반 쾌적도 분류 모델(K-Means), A* 경로 탐색 알고리즘을 융합하여, 기존 내비게이션이 제공하지 않는 **폭염 대응형 경로 안내**를 실현합니다.

---

## 2) 개발 결과물의 기능 및 특징

### 서비스의 목적 및 배경

- 국내 폭염 일수는 지속적으로 증가하고 있으며, 야외 보행·자전거 이동 시 직사광선 노출로 인한 온열질환 위험이 높아지고 있습니다.
- 기존 내비게이션(카카오맵, 네이버지도 등)은 최단거리·최단시간 중심 경로만 제공하며, 그늘 여부를 전혀 고려하지 않습니다.
- Shadow-Nav는 **"같은 시간 내 가장 시원한 경로"** 를 목표로, 도심 보행자와 자전거 이용자의 폭염 피해를 실질적으로 줄이는 서비스를 제공합니다.

### 서비스의 핵심 기술, 기능 및 구성

| 기능 | 설명 |
|------|------|
| **시간대별 그림자 맵 시각화** | 09:00~18:00, 5분 단위 총 109장의 그림자 GeoJSON을 카카오맵 위에 실시간 오버레이 |
| **그늘 최적 경로 탐색** | OSMnx 도로망 그래프에 A* 알고리즘 적용, 그림자 덮힘 비율을 가중치로 반영한 경로 계산 |
| **그늘 가중치 산정** | 도로 엣지별 그림자 GeoJSON 교차 비율 계산 → shadow_weight 부여 |
| **K-Means 쾌적도 분류** | 엣지 특성(그림자비율, 직사광선노출거리, 보도여부)을 StandardScaler 정규화 후 K-Means 클러스터링, Silhouette Score로 최적 K 자동 선택 → 등급별 배율 적용 |
| **실시간 신호등 패널티** | 공공데이터포털 교통안전 신호등 API로 경로상 빨간불 잔여시간을 라우팅 패널티에 반영 |
| **따릉이 연동** | 자전거 모드 선택 시 출발지 반경 1km 내 공영자전거 대여소 및 실시간 잔여 대수 표시 |
| **횡단보도 제약** | 서울 횡단보도 위치 데이터(Shapely 교차 분석)로 무단횡단 구간 패널티 적용, 현실적인 경로 생성 |
| **가중치 모드 선택** | 최대 그늘(max_shadow) / 균형(balance) / 최단 시간(fastest) 3가지 모드 제공 |

## 3) 기존 서비스와 차별성 및 독창성

### 기존 유사 서비스와의 차별점

| 구분 | 카카오맵 / 네이버지도 | Shadow-Nav |
|------|----------------------|------------|
| 경로 기준 | 최단시간 / 최단거리 | 그늘 비율 + 시간 복합 최적화 |
| 그림자 반영 | 없음 | 시간대별 3D 건물 그림자 GeoJSON 반영 |
| 신호등 정보 | 정적 교통정보 | 실시간 보행자 신호 잔여시간 반영 |
| 자전거 연동 | 없음 | 실시간 공영자전거 대여소 및 잔여 대수 |
| 경로 가중치 | 거리/시간 | 그늘+신호패널티+무단횡단 제약 복합 가중치 |

### 독창적 기술

- **3D 건물 기반 그림자 사전 계산:** 국토교통부 V-World 3D 건물 통합정보를 활용해 시간대별(109장) 그림자 GeoJSON을 사전 생성, 실시간 서빙으로 지연 없는 그림자 시각화 구현
- **공공 API 비동기 병렬 처리:** 신호등, 따릉이, 횡단보도 데이터를 asyncio.gather로 동시 호출해 응답 지연 최소화
- **분(minute) 단위 A* 휴리스틱:** Haversine 거리를 이동속도로 나눠 shadow_weight와 동일 단위의 휴리스틱 사용, 그늘·신호 가중치를 정확히 반영한 최적 경로 탐색
- **횡단보도 스냅 & 무단횡단 패널티:** Shapely 교차 분석 기반으로 실제 횡단보도 위치에만 보행자가 도로를 건너도록 강제하여 현실적인 경로 생성

| 데이터명 | 출처 | 획득 방법 | 활용 목적 |
|---------|------|---------|---------|
| **보행자/자전거 도로망 그래프** | OSMnx (OpenStreetMap) | Python 라이브러리 (런타임 동적 다운로드) | 길찾기 요청 bbox 기준 보행 도로망 그래프 실시간 구성, A* 라우팅 적용 |

| **카카오 Local 검색 API** | Kakao Developers | REST API (앱 키) | 장소명 → 위경도 변환, 자동완성 검색 |

### 데이터 획득 지속성 및 가공

- 그림자 GeoJSON: V-World 건물 데이터 기반 사전 계산 → 서버 내 정적 파일로 저장, 연 1회 이상 갱신 가능
- 실시간 공공 API: 공공데이터포털 정식 서비스키 발급, 페이지별 병렬 호출로 전체 데이터 수집
- 횡단보도 노드: 서버 시작 시 전체 로드 후 메모리 캐싱, 이후 Bounding Box 필터링으로 고속 응답

## 5) 개발 결과물의 기술성과 구현 및 실현가능성

### 주요 기술 스택

| 영역 | 기술 |
|------|------|
| **Frontend** | HTML5, CSS3, Vanilla JS, 카카오맵 JS API |
| **Backend** | Python 3.11, FastAPI, httpx (비동기 HTTP) |
| **라우팅 모델** | OSMnx, NetworkX (A*), GeoPandas, Shapely, Scikit-learn (K-Means 쾌적도 분류) |
| **데이터 포맷** | GeoJSON (그림자), JSON (API 응답) |

### 시스템 구조

```
사용자 브라우저 (카카오맵 UI)
    ↓ 장소 검색
카카오 Local API → 위경도 변환
    ↓ 길찾기 요청 (출발/도착 좌표, 시간, 모드)
FastAPI 백엔드
    ├─ 카카오 모빌리티 API → 기본 경로/Bbox 획득
    ├─ 신호등 API + 따릉이 API + 횡단보도 API (병렬 호출)
    └─ route_model (OSMnx 도로망 + shadow_weight + A* 탐색)
           └─ 그림자 GeoJSON 109장 (로컬 파일)
    ↓ 최적 경로 + 따릉이 대여소 + 신호 정보 응답
사용자 브라우저 → 경로 폴리라인 + 그림자 폴리곤 렌더링
```

### 구현 완성도 및 실현 가능성

- **현재 구현 완료:** FastAPI 서버, 카카오맵 UI, 그림자 시각화, A* 라우팅, 신호등·따릉이·횡단보도 API 연동, 가중치 모드 선택 기능
- **실행 가능 프로토타입:** 로컬 환경(`localhost:3000`)에서 즉시 구동 가능, API 키 설정 후 실서비스 전환 용이
- **확장성:** 서울 외 타 지자체 건물 데이터 추가 시 전국 서비스 확장 가능, 공영자전거 데이터는 이미 전국 단위 API 사용 중
- **AI 가점 항목:** Scikit-learn K-Means 비지도학습 기반 도로 쾌적도 자동 등급 분류 (StandardScaler 정규화 + Silhouette Score 최적 K 선택) → A* 라우팅 가중치에 실시간 반영

## 6) 기대 효과

### 사회적 가치 창출

- **온열질환 예방:** 폭염 취약계층(노인, 어린이, 임산부)의 야외 이동 시 직사광선 노출 최소화로 온열질환 위험 감소
- **친환경 이동 촉진:** 보행·자전거 이동의 쾌적성 향상으로 자가용 대비 도보/자전거 이용률 증가 → 탄소 배출 저감 기여
- **공영자전거 활성화:** 실시간 따릉이 대여소 연동으로 공공 자전거 인프라 활용도 제고
- **스마트시티 연계:** 공공 교통 신호·자전거 데이터와 지도 서비스의 융합으로 도시 보행 환경 데이터 활용 모델 제시

### 파급 효과

- 기후위기 대응 및 도시 보행 환경 개선 정책과의 연계 가능성
- 지자체 스마트 보행 안내 시스템으로의 확장 및 연동 가능
- 전국 통합 그늘길 데이터베이스 구축의 초석
